Aide PDFCreator
 
 

Page vide avec des caractères Unicode dans le nom de fichier

Utiliser des caractères Unicode dans le titre ou le nom de fichier génèrera des erreurs et peut produire un PDF ne contenant qu'une page blanche.

La version en cours de PDFCreator est développée avec Visual Basic 6. Les contrôles de l'interface utilisateur n'acceptent pas les chaînes unicode. Pour cette raison, PDFCreator ne peut afficher les caractères chinois que sur un système d'exploitation anglais, si le programme et la page de code sont paramétrés correctement. Malfré tout il esubsiste des problèmes pour convertir les chaînes (i.e. Title ou filename) avec les symboles internes.

C'est une raisons pour lesquelles nous avons décidé de redévelopper PDFCreator sur une plateforme .Net. Sous .Net, les contrôles acceptent totalement unicode et nous n'avons de problème pour prendre en charge les caractères chinois sous n'importe quel système d'exploitation.
