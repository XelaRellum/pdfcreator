Document Properties

The document properties section allows the user to set some generic properties for use in all PDF's created by PDFCreator.

Use Standard Author: When this item is checked, it allows the user to set a standard author that will be used every time a PDF is created. It also adds a author token to the author's name, such as Computername, DateTime, Title, Username, ClientComputer, and the RedMon tokens. Please see the Auto-Save section for more information on the tokens.

Use the current Date/Time for 'Creation Date': When checked, this sets the current date and time in the creation date instead of the date and time the document was originally created.

One page per file: This setting allows the user to create a new file for every page in the original document. Due to a restriction in Ghostscript, this does not work for PDF and EPS file formats.

Stampstring: When a text string is placed in this setting, PDFCreator puts a "stamp" in the center of each page of the document with that text. You can change the stamps font, color, size, and outline by using the following settings. WARNING: This setting is very experimental. When this setting is used, some pages of the document can become corrupted. Make sure to check your PDF after you create it.

Font-color: Changes the font color for the StampString setting.

Use outline font: Changes whether or not an outline is used for the text instead of full color text in the StampString setting.
Properties:
0 - Off (Default)
1 - On

Outline font thickness: If the StampUseOutlineFont is enables, changes the font thickness for the outline font.
