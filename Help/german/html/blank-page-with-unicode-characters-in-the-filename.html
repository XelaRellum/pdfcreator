PDFCreator Hilfe
 
 

Leere Seite, wenn Unicodezeichen im Dateinamen vorkommen

Wenn Unicodezeichen im Titel oder Dateinamen verwendet werden, kann es sein, dass Fehler auftreten oder eine leere PDFs erzeugt wird.

Die aktuelle Version des PDFCreators wird mit der Entwicklungsumgebung VB6 entwickelt. Die hier zur Verfügung gestellten Steuerelemente unterstützen leider keine Unicode Zeichenketten. Dies hat zur Folge, dass der PDFCreator auf einem deutschen Betriebssystem nur dann in chinesisch dargestellt werden kann, wenn in den Einstellungen zur Programmschiftart zusätzlich ein Zeichensatz eingestellt wird.
Jedoch gibt es weiterhin Probleme beim Auslesen Zeichenketten wie dem Titel und dem Dateinamen falls diese Unicode Zeichen enthalten.

Aus diesem Grund und noch vielen weiteren wurde die Entwicklung eines PDFCreators auf der basis von .Net begonnen. Innerhalb der .Net Umgebung sind alle Steuerelemente Unicode fähig sind. Somit gibt es dann keine Probleme mehr z.B. chinesische Zeichen auf einem deutschen Betriebssystem anzuzeigen und zu verarbeten.
